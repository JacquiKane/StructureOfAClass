Structure Of A Class : The SoccerPlayer

A SoccerPlayer model class should represent any soccer player, with instance data of name, age, position and club. The object, instantiated from the class should have callable methods, to access and change the value of the data-members. Other methods may be necessary to round out the digital model of a soccer player.

public class SoccerPlayer {
The public accessibiliy modifier makes the class visible outside of the module.
The class declaration is contained within { ... }
private String name;
private double age;
private String position;
private String club;
Access to attributes should be kept internal to the class. Therefore instance variables are designated as private.
public SoccerPlayer() {
}

public SoccerPlayer(String name, double age, String position, String club) {
 this.name = name;
 this.age = age;
 this.position = position;
 this.club = club;
}
Constructors are designated public. Constructors are used to set the initial state of an object, which should include initial values for instance variables. When no constructor is written, Java provides a no-parameter constructor, and the instance variables are set to default values.
public String getName() {
 return name;
}

public double getAge() {
 return age;
}

public String getPosition() {
 return position;
}

public String getClub() {
 return club;
}
An accessor method allows other objects to obtain the value of instance variables or static variables. The return keyword is used to return the flow of control to the caller. Note that the return type or class mirrors the type of the returned data, which is the value of the instance data member.
public void setName(String playerName) {
 name = playerName;
}

public void setAge(double playerAge) { {
 age = playerAge;
}

public void setClub(String playerClub) {
 club = playerClub;
}

public void setPosition(String playerPosition) {
 position = playerPosition;
}
A mutator method is often a void method that changes the value of an instance variable or a static variable.
public void nextGame() {

}

Non-void methods pass back data to the caller, void methods do not. Overloaded methods have the same method signature, but a different number of parameters.
} //end class definition
